# 1. Spring의 핵심 설계: 계층형 아키텍쳐

## 스프링 계층형 아키텍처
웹 브라우저 - 컨트롤러 - 서비스 - DAO - DB
DTO(Data Transfer Object): 데이터 전송 객체
Entity: DB 테이블과 맵핑되는 핵심 객체, 외부 직접 노출 금지

# 2. 웹 요청의 진입점: 컨트롤러 계층
모든 Client의 요청이 컨트롤러로 들어옴. 컨트롤러는 Request를 처리할 수 있는 서비스한데 전달해주는 역할을 해줌.

# 3. 스프링 빈 & 의존성 주입

스프링 빈: 스프링 컨테이너 안에 담겨져 있다. 어플리케인션 전역에서 사용할 공용객체이다.

의존성 주입: 내가 의존하는 객체를 직접 생성하지 않고 밖에서 주입 받는 것, 생성자 주입으로 설정

# 4. 비즈니스 로직이 중심: 서비스 계층
컨트롤러가 전달해준 request 중 실질적인 비즈니스 로직을 담당하는 역할을 한다. 필요하면 DB로 보내서 원하는 답을 갖고온다.
